# ATADA — Production Migration Checklist

> This document lists every mock/stub that must be replaced before deploying to production.

---

## 1. DATABASE

| File | Current (Mock) | Production Replacement |
|------|---------------|----------------------|
| `backend/.env` → `DATABASE_URL` | `sqlite:///./atada.db` | `postgresql://user:pass@host:5432/atada` |
| `backend/app/db/database.py` | `connect_args={"check_same_thread": False}` | Remove this line (PostgreSQL doesn't need it) |
| `backend/seed_mock_db.py` | Faker-generated data | Real data from parser service |

**Action:** Set up PostgreSQL (Neon, Supabase, or Railway), run Alembic migrations, connect parser pipeline.

---

## 2. AUTHENTICATION — SMS OTP

| File | Current (Mock) | Production Replacement |
|------|---------------|----------------------|
| `backend/app/services/auth.py` → `send_otp()` | `print()` to console | Twilio / MessageBird API call |
| `backend/.env` | No Twilio keys | Add `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_PHONE_NUMBER` |

**Mock code location (lines 29-36 in auth.py):**
```python
# MOCK: Print to console instead of sending real SMS
print(f"ATADA OTP for {phone}: {code}")
```
**Replace with:** Twilio `client.messages.create(to=phone, body=f"Your Atada code: {code}")`.

---

## 3. AI / LLM CHAT

| File | Current (Mock) | Production Replacement |
|------|---------------|----------------------|
| `backend/app/services/chat.py` → `_stream_mock()` | Hardcoded responses from `MOCK_RESPONSES` list | Automatic: set `MINIMAX_API_KEY` in `.env` |
| `backend/.env` → `MINIMAX_API_KEY` | Empty string (triggers mock) | Real MiniMax API key |
| `backend/.env` → `MINIMAX_GROUP_ID` | Empty string | Real MiniMax group ID |

**How it works:** The `stream_chat_response()` function auto-detects: if `MINIMAX_API_KEY` is set, it uses the real MiniMax API (`_stream_minimax`). If empty, it falls back to `_stream_mock`. No code changes needed — just set the env var.
