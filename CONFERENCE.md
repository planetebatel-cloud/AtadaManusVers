# ATADA — Conference Day Checklist

> Что делать в день показа, чтобы ничего не упало на сцене.

---

## 30 минут до выступления

### 1. Прогреть бэкенд (Render free спит после 15 мин)

```bash
curl https://atada-api.onrender.com/api/health
# Ждать ответ {"status":"ok"} — первый раз может занять ~30-40 сек
```

Если ответ есть и быстрый (<1 сек) — бэк уже тёплый. Если нет — подожди и повтори.

### 2. Открыть live-сайт на устройстве показа

https://atada.vercel.app

Проверь глазами:
- [ ] Сверху видна чёрная полоска "ATADA · Live demo · Try demo"
- [ ] Job-карточки видны (Wix, Monday.com, etc.) с картинками, не серые
- [ ] Если есть онбординг-оверлей — нажми Skip intro один раз (он запомнит)

### 3. Пройти one-click демо ОБА раза

`/auth` → **Worker demo** → должен залогинить и редиректнуть на `/`.
Свайпни вправо одну карточку — посмотри что не падает.

`/auth` → **Employer demo** → должен пойти на `/employer` со статами.

### 4. Очистить локальное состояние перед показом

Если хочешь чтобы зрители увидели свежий онбординг:
```javascript
// В DevTools Console на atada.vercel.app:
localStorage.clear()
location.reload()
```

## Что показывать (story arc, 3-5 минут)

### Сценарий "Worker"
1. Открой `atada.vercel.app` — "это новый job-board для Израиля, заточенный под мобайл и AI"
2. Покажи демо-баннер — "любой может зайти без телефона и попробовать"
3. `/auth` → **Worker demo** — "один клик, никаких смс"
4. На главной — "вот AI-ранжированный фид: матч-скор, время в дороге на машине и на общественном транспорте, зарплата"
5. Свайпни Apply на топовой вакансии — "это записалось в applications, реальная база"
6. Открой `/applications` — "вот трекинг откликов"
7. Открой `/profile` — "резюме редактируется, PDF скачивается с watermark"

### Сценарий "Employer"
1. Логаут из Worker → `/auth` → **Employer demo**
2. `/employer` — "вот дашборд: открытые вакансии, кандидаты с trust-score, статус найма"
3. (Опционально) Перейди в `/pricing` — "монетизация: 199 ILS за пост, 999 ILS/мес подписка. Stripe в test mode"

### Что НЕ показывать (или сразу пояснять)
- ❌ AI чат — он сейчас мокает ответы. Если откроют чат — скажи "MiniMax integration в финальной фазе, ответы пока заранее заготовленные". Не вводи в чат сам если не нужно.
- ❌ Реальный SMS — если кто-то попробует логин со своим номером, OTP уйдёт в Render console и никуда больше. Веди людей через demo-кнопки.
- ❌ Stripe — кнопки работают, но "это test-mode, реальная карта будет отвергнута".

## Если что-то упало во время демо

### Симптом: сайт не открывается / белый экран
1. **Открой DevTools → Network**. Что красное?
2. Если 502/503/504 на `/api/...` → бэк уснул, открой `https://atada-api.onrender.com/api/health` в новой вкладке, дождись 200 OK, обнови основную.
3. Если CORS-ошибка → проверь `CORS_ORIGINS` env var в Render (должен быть `https://atada.vercel.app`).
4. Если 404 на статике → проверь Vercel — последний deployment должен быть READY на https://vercel.com/planetebatel-5775s-projects/atada

### Симптом: job-карточки серые без картинок
Backend не отдаёт `image_url`. Либо auto-seed не отработал (БД пустая), либо
Unsplash блочит. Запасной план: показывай функционал через Swagger
https://atada-api.onrender.com/api/docs — там видна вся API-поверхность.

### Симптом: Demo Worker button не работает
1. Проверь `/api/auth/otp/peek?phone=%2B972501234567` напрямую — должен вернуть `code`.
2. Если 403 — backend перезапустился и потерял хардкод DEMO_PHONES. Проверь
   `backend/app/api/auth.py:DEMO_PHONES` — должны быть оба номера.
3. Запасной план: на `/auth` вручную введи `+972501234567`, дождись отправки OTP,
   потом скопируй код из Render logs или сделай curl на `/otp/peek`.

### Симптом: вообще всё мёртвое
Локальный fallback: открой ноут, запусти бэк + фронт локально, покажи
`http://localhost:3000` с проектора. Всё работает идентично.

```powershell
# Терминал 1
cd D:\AtadaManusVers\backend
python -m uvicorn app.main:app --port 8002

# Терминал 2
cd D:\AtadaManusVers
pnpm dev
```

## Что доделать перед следующим показом (по приоритету)

1. **MiniMax AI chat** — переписать `backend/app/services/chat.py:_stream_minimax`
   под Anthropic-compatible endpoint Token Plan. Нужны от пользователя:
   точный URL и название модели. ~20 мин работы.
2. **Прогрев бэкенда автоматический** — cron-job.org или GitHub Action который
   пингует `/api/health` каждые 10 минут. Бесплатно, бэк не спит.
3. **Postgres вместо SQLite** — Neon free tier, applications перестанут
   сбрасываться при каждом рестарте Render.
4. **Twilio для реальных SMS** — если хочется чтобы зрители могли логиниться
   со своими номерами после показа.
